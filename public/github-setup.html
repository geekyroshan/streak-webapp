GitHub OAuth Setup Guide

Note: This page is a setup guide to help you configure your GitHub OAuth credentials for this application.

Quick Fix Steps

Step 1: Create a GitHub OAuth App

You need to create an OAuth App in your GitHub account:

  1. Go to GitHub Developer Settings
  2. Click "OAuth Apps" then "New OAuth App"
  3. Fill in the details:
    • Application name: GitHub Streak Manager (or your preferred name)
    • Homepage URL: https://your-vercel-domain.vercel.app
    • Authorization callback URL: https://your-vercel-domain.vercel.app/api/auth/github/callback
  4. Click "Register application"
  5. After creation, you'll see your Client ID. Click "Generate a new client secret" to generate a secret.

Step 2: Configure Environment Variables in Vercel

Go to your Vercel project settings and add these environment variables:

GITHUB_CLIENT_ID=your_client_id_here
GITHUB_CLIENT_SECRET=your_client_secret_here
VITE_GITHUB_CLIENT_ID=your_client_id_here  # Same as above, for frontend access

Step 3: Redeploy Your Application

After setting the environment variables, trigger a new deployment in Vercel:

  1. Go to your Vercel dashboard
  2. Select your project
  3. Click "Deployments" tab
  4. Click "Redeploy" on your latest deployment

Troubleshooting

  • Check that your GitHub OAuth App's callback URL matches your Vercel domain exactly
  • Verify that you've set all required environment variables
  • Test the API health endpoint: /api/health

Important: The GitHub login will not work until you complete these steps!
